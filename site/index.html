Chạy Runtime là runtime chính sách dựa trên ghi chú (note-based) cho các CLI code đa tác tử. Tạo work note, dispatch worker, validate result_note, chạy test tuỳ chọn và kiểm tra patch boundary trước khi báo done.

$ npm install -g chay-runtime
Bắt đầu ngay Xem console realtime Xem trên npm
Vì sao chọn Chạy Runtime

Ranh giới runtime giữa tác tử và mã nguồn của bạn

Tác tử đọc các ghi chú JSON gọn nhẹ, con người đọc bản kiểm toán Markdown, và mọi bản vá đều được kiểm tra trước khi áp dụng.

Ngữ cảnh dựa trên ghi chú

Tác tử đọc ghi chú JSON gọn từ memory/*.json. Con người xem Markdown từ audit/*.md. Không rò rỉ log thô.

Kiểm tra ranh giới

Kiểm tra kích thước ghi chú, schema đầu ra, kích thước bản vá và phạm vi file. Sửa ngoài phạm vi và anti-pattern đều bị từ chối.

Trí tuệ về repo

Chọn sẵn một gói ngữ cảnh nhỏ trước khi tác tử đọc mã, giúp worker tập trung và tiết kiệm.

Quy tắc kiến trúc

Worker phải tuân theo các mẫu thiết kế sẵn có và nguyên tắc SOLID — được cưỡng chế, không chỉ gợi ý.

Nén trải nghiệm

Bộ nhớ, kỹ năng và quy tắc được phân tầng để ghi chú công việc luôn nhỏ gọn với ngân sách token có giới hạn.

Giao diện quy trình gốc

Chạy cr ui serve để tạo task, dispatch worker, validate result, patch check, token/eval report, chat và xem SSE realtime.

Bắt đầu nhanh

Thiết lập trong dự án của bạn

Cài đặt toàn cục, rồi chạy setup ngay trong repo bạn muốn tác tử làm việc.

terminal 
# 1 · cài bộ công cụ
npm install -g chay-runtime

# 2 · thiết lập ngay trong dự án của bạn
cd du-an-cua-ban
cr setup --agents claude,codex,antigravity --main codex

# 3 · tạo task compact và giao cho worker
cr task "Sửa lỗi apply job bị trùng" --compact --max-notes 2

# 4 · chạy worker, tự ghi progress + history realtime
cr dispatch codex --agent=codex --max-retries 3 \
  --test-command "npm test"

# hoặc mở console rồi bấm Create + Run
cr ui serve --port 7770
Tài liệu lệnh

Toàn bộ lệnh CLI, khớp 1-1 với mã nguồn

Đây là toàn bộ lệnh thật của cr (xem cr help hoặc src/utils/help.js), không phải ví dụ minh hoạ.

Khởi tạo & thiết lập cr init cr setup --agents claude,codex,antigravity --main codex cr doctor

Tạo cấu trúc thư mục dự án, hỏi/nhận danh sách agent rồi ghi memory/host_config.json. --main nhận claude, codex hoặc antigravity; nếu bỏ --workers, các agent còn lại tự làm worker.

Task nhanh cr task "Sửa lỗi apply job bị trùng" --compact --max-notes 2

Gộp việc tạo task_note, context_package và work_note vào một lệnh duy nhất.

Ranh giới (Boundary) cr boundary check-note --file memory/task_note.json --kind task cr boundary validate-output --file memory/codex_result_note.json

Validate một ghi chú JSON theo schema và ngân sách token của policy; với result note còn trả về retry_instruction gọn nếu sai.

Trí tuệ repo cr repo scan --root . --out .chay-index/project_map.json cr context plan --task "..." --out memory/context_package.json

Quét repo và cache theo mtime + size, rồi chấm điểm file theo từ khóa trong task để chọn tối đa N file liên quan nhất.

Ghi chú → Markdown cr note compile --json memory/task_note.json --out audit/task_note.md

Chuyển một ghi chú JSON thành file Markdown cho con người đọc — agent không được đọc file audit này.

Kiểm tra bản vá cr patch check --diff .chay/tmp/current.diff --work memory/codex_work_note.json

Đối chiếu git diff với allowed_files và giới hạn số file/số dòng thay đổi trong policy.

Tiến độ cr progress update --agent codex --step editing --message "..." cr progress update --agent codex --step validate_result --message "..."

Ghi một mốc tiến độ vào memory/<agent>_progress.json và nối vào lịch sử _progress_history.json. Console đọc các file này realtime qua SSE.

Đo lường & báo cáo cr token report cr eval report

So sánh token ghi chú hiện tại với ước lượng cả repo để tính % tiết kiệm; và chấm điểm 0–100 cho task vừa chạy (hoàn thành, khớp work/result, patch đúng phạm vi, tín hiệu test, số lần retry).

Gói việc cr workpack make --worker codex --goal "..." --compact --out memory/codex_work_note.json

Tạo work_note JSON gọn cho một worker: allowed_files, quy tắc kiến trúc, và output_contract.

Điều phối cr dispatch codex --agent=codex --max-retries 3 cr dispatch codex --agent=codex --test-command "npm test"

Chạy worker thật, ghi progress từng bước, validate + retry result_note, chạy test command nếu có, rồi kiểm tra patch boundary trước khi báo "done".

Nén trải nghiệm cr experience snapshot --out memory/experience_spectrum.json

Xuất bản tóm tắt memory/skills/rules đang dùng theo khung Experience Compression Spectrum.

Tích hợp cr integration install --target claude|codex|antigravity

Copy template tích hợp vào dự án: subagent Claude, hoặc file hướng dẫn cho Codex/Antigravity.

Mô hình an toàn

Mặc định là từ chối

Chạy Runtime "fail closed". Nếu tác tử bước ra ngoài hợp đồng, runtime sẽ chặn lại.

  • Ghi chú quá dài
  • Ghi chú kết quả không khớp schema
  • Bản vá đổi quá nhiều file
  • Bản vá thêm quá nhiều dòng
  • Sửa file ngoài danh sách cho phép
  • Tác tử đọc Markdown kiểm toán
  • Ghi chú thiếu quy tắc kiến trúc / SOLID
  • Log kết quả tự do quá lớn
